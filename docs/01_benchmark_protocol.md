### 2.2 Pose Selection Strategy

- Pose selection: **rank1 only**
- Definition:
  - The highest-confidence predicted pose per ligand is used.
  - For each ligand, DiffDock generates multiple pose hypotheses via stochastic sampling. 
  Each pose is assigned a confidence score by the trained confidence model. 
  The pose with the highest confidence score is selected as the rank1 pose.
- Rationale(why only Rank1?):
  - Ensures consistency between docking ranking and virtual screening metrics.
  - To ensure consistency between docking prediction and virtual screening ranking, only the highest-confidence pose per ligand is used.

---

### 2.3 Confidence Score Definition

- Confidence score extracted from:
  - diffdock_scores_rank1.csv, generated by make_diffdock_score_table.py based on the rank1 pose outputs.
- Ranking criterion:
  - Ligands sorted by descending confidence score.
- Used for:
  - ROC-AUC
  - EF@k%
  - nEF
  - Calibration analysis

## 3. Inference Stability Definition

### 3.1 Status Categories

Each ligand is categorized as:

- **Success**
- **Skip**
- **Fail**

Definitions:

- Success:
  - Rank1 pose file exists
  - Confidence score successfully computed
  - Successfully generated rank1 pose and confidence score
- Fail:
  - Inference terminated due to graph construction failure
  - Condition: ["No edges and no nodes..." or equivalent]
  - No valid pose generated
- Skip:
  - Ligand not processed in either test or confidence dataset
  - Condition 1: ["test dataset does not contain..."]
  - Condition 2: ["confidence dataset does not contain..."]


Parsing method:

- Script used: `parse_inference_err.py`
- Source: `.err` log files per target

---

### 3.2 Retry Definition

- Retry = Fail + Skip
- Retry rate per target:
  - `retry_rate = (fail + skip) / total_ligands`

---

### 3.3 Missing Policy

- missing_policy = **bottom**
- Definition:
  - Missing ligands (skip/fail)are assigned the lowest possible confidence score and ranked at the bottom of the list.
  - artificial inflation 방지를 위해 missing ligand는 ranking에서 가장 마지막에 위치하도록 하는 방침.
- Rationale:
  - Conservative evaluation strategy
  - Avoids artificial performance inflation

## 4. Structural Validity Criteria

### 4.1 COM Distance Criterion

- Metric: Center-of-Mass (COM) distance
- Definition:
              COM= | C_predicted​−C_reference ​|
  - Why COM distance:
      - RMSD보다 계산이 안정적
      - atomic ordering mismatch 문제 회피
      - coarse pose correctness 판단에 충분
- Threshold: **COM ≤ 2 Å**
- Interpretation:
  - Pose considered structurally valid if COM ≤ 2 Å relative to reference
- Script used:
  - `compute_comdist2.py`
